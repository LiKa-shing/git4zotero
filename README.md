# git4zotero 使用指南

## 插件简介

`git4zotero` 是一款面向 Zotero 8/9 的论文附件版本管理插件。它使用本机 Git 为 Zotero 条目中的 Word 论文附件建立本地版本历史，支持创建版本、检查修改、查看历史摘要和恢复旧版本。

当前插件信息：
- 兼容范围：Zotero `8.0` 到 `9.0.*`
- 推荐格式：`.docx`
- 支持格式：`.docx`、`.doc`


## 安装前准备

1. 安装 Zotero 8 或 Zotero 9。
2. 安装 Git，并确保系统可以执行：
   ```powershell
   git --version
   ```

如果 Git 没有加入系统 PATH，可以稍后在 Zotero 的插件设置页中手动填写 `git.exe` 路径，例如：

```text
C:\Program Files\Git\cmd\git.exe
```

## 安装插件

1. 打开 Zotero。
2. 进入 `工具 -> 插件`。
3. 将 `git4zotero.xpi` 拖入插件窗口。
4. 或点击插件窗口中的齿轮菜单，选择 `Install Add-on From File...`。
5. 安装完成后重启 Zotero。


## 初始配置

重启 Zotero 后，进入设置页：

- Windows/Linux：`编辑 -> 设置`
- macOS：`Zotero -> 设置`

找到“论文版本管理”设置页，配置以下项目：

- `Git 可执行文件路径`：填写 Git 路径。留空时插件会尝试使用系统 PATH 中的 `git`。
- `测试 Git`：测试当前 Git 路径是否可用。
- `默认版本说明`：创建版本时如果说明为空，将使用此默认文本。
- `恢复旧版本前自动创建安全版本`：建议保持开启。恢复旧版本前，插件会先保存当前文件状态，降低误覆盖风险。

## 基本使用流程

1. 在 Zotero 条目中添加论文附件，推荐使用 `.docx`。
2. 在条目列表中右键该 Zotero 条目，或右键对应论文附件。
3. 打开 `论文版本` 菜单。
4. 第一次使用时，点击 `启用版本管理`。
5. 启用后，点击 `创建版本...`，填写版本说明，例如：
   ```text
   初稿
   ```
6. 在 Word、WPS 或 LibreOffice 中编辑论文。
7. 编辑完成后，务必先在文字处理软件中保存文件。
8. 回到 Zotero，右键同一条目或附件，选择 `检查修改`。
9. 确认需要保存当前修改时，选择 `创建版本...`。
10. 需要回退时，选择 `恢复版本...`，再从列表中选择目标版本。

## 右键菜单说明

`论文版本` 菜单包含以下操作：

- `启用版本管理`：为当前条目开启版本控制。
- `检查修改`：比较当前论文文件与上一版本的差异。
- `创建版本...`：将当前论文文件保存为一个新版本。
- `恢复版本...`：选择历史版本并恢复。
- `配置 Git 路径...`：当 Git 不可用时快速配置路径。
- `停用版本管理`：停止当前条目的版本管理，不删除已有版本历史。

插件只支持单选操作。一次选择多个 Zotero 条目时，版本操作不可用。

## 右侧“论文版本”面板

选中 Zotero 条目后，右侧条目面板会显示“论文版本”区域，用于查看当前状态。

常见状态包括：

- 未找到可管理的论文文件
- 尚未启用版本管理
- Git 不可用
- 已启用，显示版本数量、最近检查结果和 Git 工作树状态
- 版本历史摘要

注意：右侧面板主要用于显示状态。启用、创建、检查、恢复等操作应通过条目列表右键菜单完成。

## `.docx` 与 `.doc` 的差异

`.docx` 支持正文级识别。插件会读取正文、脚注和尾注，排除删除修订和批注，并记录页眉页脚上下文。检查修改时可以显示新增、删除、修改段落以及字数变化。

`.doc` 仅支持文件级识别。插件可以判断文件是否变化，并创建或恢复版本，但不会展示具体段落差异。

## 恢复旧版本

恢复流程如下：

1. 右键已启用版本管理的条目或附件。
2. 选择 `论文版本 -> 恢复版本...`。
3. 从版本列表中选择目标版本。
4. 确认恢复。

默认情况下，插件会在恢复前自动创建一个“恢复前自动备份”版本。建议保持该设置开启。

## 使用建议

- 优先使用 `.docx`，可以获得更准确的正文差异识别。
- 每次编辑后先在 Word/WPS/LibreOffice 中保存，再回到 Zotero 检查或创建版本。
- 版本说明建议写清楚本次修改内容，例如“完成方法章节修改”。
- 不要手动修改插件数据目录中的 Git 仓库和 `.git4zotero` 元数据。
- 恢复旧版本前确认当前文件已经关闭或已保存，避免文字处理软件覆盖恢复结果。

## 常见问题

如果提示 `Git 不可用`：

1. 确认已安装 Git。
2. 在命令行运行 `git --version`。
3. 在 Zotero 设置页填写完整 `git.exe` 路径。
4. 点击 `测试 Git`。

如果找不到“论文版本”菜单：

1. 确认只选中了一个条目或附件。
2. 确认该条目存在 `.docx` 或 `.doc` 附件。
3. 重启 Zotero。
4. 确认安装的是最新版本的 `git4zotero.xpi`。
